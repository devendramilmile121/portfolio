Fundamentals

What is GitHub Copilot?

  • AI pair programmer — real-time code suggestions inside your IDE
  • Powered by LLMs trained on billions of lines of public code
  • Works in VS Code, Visual Studio, JetBrains, Neovim
  • Context-aware — reads your workspace, types, and imports

Why It Matters

Removes friction from boilerplate. You focus on architecture and logic, not syntax.

Prompting

Intent-Driven Prompts

Vague
// fetch data async function getData() { const res = await fetch('/api'); return res.json(); }
Clear
// Fetch user by ID with error handling // - Use AbortController for timeout // - Return UserData | null // - Handle 404 gracefully export async function fetchUser(id: string) { // Copilot handles implementation }

The key: Describe WHAT and WHY. Let Copilot figure out HOW. Comments are your power tool.

Context

Mastering Context

Wasting Tokens

  • 20+ unrelated tabs open
  • Using @workspace for every question
  • Massive monolith files
  • Ignoring .copilotignore

Smart Context

  • 2-3 related files open only
  • Use #file or #selection
  • Maintain .copilotignore
  • Keep examples visible in tabs
Tools

Using the Right Tool

Inline Suggestions

Best for: Boilerplate, test scaffolding, interfaces. Accept with Tab.

Copilot Chat

Best for: Reasoning, refactoring, explaining. Use #file for context.

Inline Fix

Best for: Debugging compiler errors. Click the lightbulb.

CLI: gh copilot

Best for: Terminal — docker, bash, git commands.

Configuration

Custom Instructions

User Level

VS Code Settings. Your personal coding style & preferences.

Workspace Level

.github/copilot-instructions.md. Team architecture & standards.

Chat Level

Dynamic context in specific threads. Override rules on-the-fly.

Pro Tip: Always create .github/copilot-instructions.md in your repo for consistent team-wide guidance.

Testing

Test Generation Strategy

Unit Tests

// Test calculateTotal: // - standard cart // - empty cart // - discount applied test('standard cart', () => { expect(calculateTotal([])) .toBe(30); });

Integration Tests

// Mock database, test API // - database returns user // - endpoint returns 200 // - response matches schema it('returns user data', async () => { db.mockResolvedValue({ id: '1' }); const res = await request(app); });
Advanced

Agents & Plan Mode

  • Agent Mode (Copilot Edits): Modifies multiple files simultaneously
  • Plan First: Never let agents edit blindly. Ask for a Markdown plan
  • Human-in-the-Loop: Review & approve before code is written

Example: "@workspace I need to add Redis caching. Generate a step-by-step plan first. Which files will you touch?"

### Plan: Redis Caching 1. Update .env for REDIS_URL 2. Create src/utils/redisClient.js 3. Modify src/services/dataService.js 4. Update routes to invalidate cache **Awaiting approval...**
Critical

Code Quality & Risks

Hallucinations — Verify imports & framework features. Don't trust made-up libraries.

Security — Deep review all auth, encryption, input sanitization. Never skip this.

Performance — Watch for N+1 queries, missing async/await, memory leaks. Profile carefully.

Code Review — Same standards as hand-written code. You own it. No exceptions.

Summary

Best Practices Checklist

Close irrelevant tabs. Keep context lean.
Write intent-driven comments — WHAT and WHY.
Define Custom Instructions (User + Workspace).
Use Plan Mode before Agents modify files.
Review every suggestion ruthlessly.
Generate tests for everything.
Match tool to task (inline vs chat vs fix).
Iterate relentlessly on prompts.
Verify security & performance manually.
You own the code. AI is just the tool.

Copilot: Force Multiplier

Master the fundamentals. Write clear prompts. Review ruthlessly. The competitive advantage isn't knowing Copilot exists — it's knowing how to use it well.
